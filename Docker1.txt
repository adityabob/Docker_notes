what is docker?
Docker is an open-source platform used to build, package, ship, and run applications inside containers. A container includes the application code, runtime, libraries, and dependencies, so it can run consistently across environments.

Think of Docker as a lightweight virtual machine alternative.
Instead of saying:

features:
1. Consistent Environment
2. Faster Deployment
3. Easy Scalability: Run multiple containers behind a load balancer.
- Horizontal scaling
- Better performance
4. Microservices Architecture:Each service can run in its own Docker container.
5. Resource Efficient:Each service can run in its own Docker container.


Docker Architecture in Detail:

Main parts:

Docker Client / CLI: Docker CLI is what users interact with.
Docker Engine:Docker Engine is the complete runtime platform. 
               -Docker Engine includes:

                 Docker Daemon
                  REST API
                  CLI interaction layer
                   Container runtime
Docker Daemon: Docker Daemon is the core background service running on host machine.(Daemon = Manager / Brain)
              -Responsibilities:
                Build images
                Run containers
                Manage networks
                Manage volumes
                Pull/push images
                Listen to Docker API requests
Images:  Docker Image contains:
          App code
          Dependencies
          Runtime
          Libraries

Containers:Docker Container is a running image.
Volumes: Docker Volume stores permanent data.
Networks: Docker Network connects containers
Docker Registry:Docker Hub stores images online.
